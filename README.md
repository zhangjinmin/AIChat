# AI 智能分析与 DAX 助手 (Power BI 视觉对象)

> 一个能让 Power 报表"开口说话"的自定义视觉对象，支持自然语言问数、智能图表生成、报表联动筛选，还能帮你写 DAX。

## 这是什么？

这是一个 Power BI 自定义视觉对象，核心目标是：**让你用说话的方式分析数据**。

它不只是一个聊天窗口，而是能：

- 读取你当前页筛选后的真实数据，基于实际业务上下文回答问题
- 自动推荐合适的图表类型并渲染展示
- 在分析结果中生成可点击的联动按钮，反向控制报表筛选器
- 帮你写符合你模型结构的 DAX 代码，减少试错成本
- 支持多级数据脱敏，适配不同安全等级的使用场景

适合零售、快消、连锁等需要频繁做品类/门店/单品下钻分析的场景。

## 主要功能

### 🔍 数据洞察 + 自动出图

- AI 读取的是**当前页面筛选后的数据快照**，不是全量数据，结论更贴合你正在看的业务场景
- 根据数据特征自动匹配图表类型，目前支持：横向条形图、雷达图、饼图/圆环图、面积图、双轴组合图
- 内置业务规则约束，避免 AI 跑题闲聊，聚焦业务问题

### 🔗 分析结果可交互（反向联动）

- 当 AI 发现异常点（比如某门店销售骤降），会在回复中生成类似 `🔍 联动分析: 门店_005` 的可点击按钮
- 点击后自动触发 Power BI 底层筛选器，报表其他图表同步过滤，实现"看到问题 → 一键下钻"

### 🔐 四级数据脱敏（按需开关）

考虑到商业数据敏感性，支持前端实时切换脱敏级别：



| 级别       | 说明                                           | 适用场景                           |
| ---------- | ---------------------------------------------- | ---------------------------------- |
| 🟢 明文     | 原始数据透传                                   | 内网测试、公开数据                 |
| 🟡 关键隐私 | 门店/供应商/业务员等实体编码，品类品牌保留明文 | 日常分析，兼顾安全与语义           |
| 🟠 全文脱敏 | 所有文本维度编码处理                           | 对外协作、跨部门共享               |
| 🔴 严格隐私 | 文本编码 + 数值标量混淆（金额/销量等）         | 高敏数据，占比/排名/趋势计算仍精确 |

> 混淆采用动态标量乘法，不改变相对关系，确保分析结论可靠。

### 💡 DAX 辅助生成（减少手写错误）

- 支持导入你通过 Tabular Editor / DAX Studio 导出的模型结构（表、列、关系、度量值）
- AI 生成 DAX 时会严格遵循你的真实模型，避免"编造表名"导致报错
- 支持识别 `USERELATIONSHIP` 等特殊关系，生成可直接使用的代码

### ⚙️ 集中配置 + 快捷指令

- 只需写一个 DAX 度量值，统一管理 API 地址、Key、模型参数等：

  ```DAX
  AI_Global_Config = 
  "{
    ""baseUrl"": ""https://api.your-llm-provider.com/v1"",
    ""apiKey"": ""sk-xxxxxx"",
    ""modelName"": ""your-model-name""
  }"
  ```

- 支持将业务分析框架封装为度量值，注入系统提示词，让 AI 按你的思路输出

## 快速开始

### 1. 准备环境

- Power BI Desktop（支持自定义视觉对象版本）
- 可用的 LLM API 服务（兼容 OpenAI 格式接口）

### 2. 导入视觉对象

- 从 `.pbiviz` 文件导入，或通过 Power BI 视觉对象市场安装（待上架）

### 3. 配置全局参数

新建度量值 `AI_Global_Config`（示例见上文），填入你的 API 信息。

### 4. 拖入画布 + 绑定数据

- 将视觉对象拖到报表页
- 绑定你需要分析的数据表或度量值
- 在设置面板选择脱敏级别等

### 5. 开始对话

在输入框用自然语言提问，例如：

- "最近 4 周汤圆品类哪个门店卖得最好？"
- "帮我看下新品引进后 30 天的动销表现"
- "写一个计算同比增幅的 DAX，考虑闰年"

## 常见问题

**Q：数据会传到外部吗？**
A：会调用你配置的 LLM 接口。建议：① 使用企业私有模型；② 开启脱敏模式；③ 避免在 Prompt 中拼接敏感字段。

**Q：支持本地部署的模型吗？**
A：只要接口兼容 OpenAI 格式（POST /v1/chat/completions），理论上都可接入，包括 Ollama、LocalAI 等。

**Q：DAX 生成不准怎么办？**
A：请确保已正确导入模型结构字典。AI 依赖你提供的元数据，不是"猜"你的模型。
